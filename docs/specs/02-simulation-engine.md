    // USD revenue per step\n  revenueGrowthRate: number;        // per-step growth rate\n  distributionModel: 'revenue-share' | 'buyback-burn' | 'hybrid';\n  /** For revenue-share: fraction distributed to token holders */\n  shareRatio: number;\n  /** For buyback-burn: fraction of revenue used for buyback */\n  buybackRatio: number;\n}\n\ninterface CompositeParams {\n  type: 'composite';\n  /** Array of sub-models composed together */\n  models: ModelParams[];\n  /** How the models interact: which outputs feed into which inputs */\n  connections: ModelConnection[];\n}\n\ninterface ModelConnection {\n  /** Index of source model in the models array */\n  sourceModelIndex: number;\n  /** Field path in the source model's state output */\n  sourceField: string;\n  /** Index of target model */\n  targetModelIndex: number;\n  /** Field path in the target model's params to override each step */\n  targetField: string;\n  /** Optional transform function name */\n  transform?: 'identity' | 'negate' | 'multiply' | 'add';\n  transformConstant?: number;\n}\n```\n\n---\n\n## 2. Simulation Algorithm Design\n\n### 2.1 Architecture: The Simulation Loop\n\nThe engine follows a discrete-time, step-forward simulation pattern. At each step, it processes emissions, fees, user behavior, reward distribution, and price discovery in a deterministic order. The order matters -- it defines the \"physics\" of the protocol.\n\n**Execution order within each step:**\n\n```\nSTEP t:\n  1. EMIT        - Mint new tokens according to emission schedule\n  2. COLLECT     - Collect fee revenue from estimated activity\n  3. BEHAVE      - Process user entries and exits\n  4. DISTRIBUTE  - Distribute rewards to stakers\n  5. PRICE       - Recalculate token price based on supply/demand\n  6. MEASURE     - Compute derived metrics (APY, runway, risk scores)\n  7. SNAPSHOT    - Record SimulationState[t]\n```\n\nThis ordering means: emissions happen before users decide whether to enter/exit (they can see current rewards), but price updates happen after all actions (reflecting the net effect of the step).\n\n### 2.2 MVP Algorithm: Staking with Emissions -- Full Pseudocode\n\n```\nFUNCTION simulateStakingEmissions(config: SimulationConfig): SimulationResult\n\n  params = config.modelParams as StakingEmissionsParams\n  behavior = config.behaviorConfig\n  scenario = config.scenario\n  T = config.timeConfig.steps\n\n  // --- INITIALIZATION (Step 0) ---\n  state = new SimulationState()\n  state.step = 0\n  state.totalSupply = params.initialTotalSupply\n  state.circulatingSupply = params.initialCirculatingSupply\n  state.totalStaked = behavior.initialUsers * behavior.averageStakeSize\n  state.tokenPrice = params.initialTokenPrice\n  state.treasuryBalanceTokens = params.initialTreasuryBalance\n  state.activeUsers = behavior.initialUsers\n\n  results: SimulationState[] = [snapshot(state)]\n  agents: AgentState[] = initializeAgents(behavior)\n\n  // Windowed accumulators for rolling metrics\n  feeWindow: number[] = []         // last 30 steps of fee revenue\n  rewardWindow: number[] = []      // last 30 steps of reward outflows\n  WINDOW_SIZE = 30\n\n  // --- MAIN LOOP ---\n  FOR t = 1 TO T:\n\n    prevState = results[t - 1]\n\n    // ========================================\n    // PHASE 1: EMIT\n    // ========================================\n    emissions = calculateEmissions(params, t, prevState.totalSupply)\n\n    // Respect max supply cap\n    IF params.maxSupply > 0:\n      emissions = MIN(emissions, params.maxSupply - prevState.totalSupply)\n      emissions = MAX(emissions, 0)\n\n    state.totalSupply = prevState.totalSupply + emissions\n    state.cumulativeEmissions = prevState.cumulativeEmissions + emissions\n\n    // Split emissions between stakers and treasury\n    stakerRewards = emissions * params.stakerEmissionShare\n    treasuryEmissions = emissions * (1 - params.stakerEmissionShare)\n\n    // ========================================\n    // PHASE 2: COLLECT FEES\n    // ========================================\n    IF params.feeModel == 'percentage-of-volume':\n      feeRevenue = params.estimatedVolumePerStep * params.feeRate\n                   * scenario.marketSentiment  // volume scales with sentiment\n    ELSE:  // flat-per-tx\n      feeRevenue = params.estimatedTxPerStep * params.flatFeeAmount\n                   * scenario.marketSentiment\n\n    state.treasuryBalanceTokens = prevState.treasuryBalanceTokens\n                                  + treasuryEmissions\n                                  + feeRevenue\n\n    // ========================================\n    // PHASE 3: USER BEHAVIOR\n    // ========================================\n    \n    // 3a. Calculate entry rate\n    baseEntry = calculateEntryRate(behavior, t)\n    sentimentAdjustedEntry = baseEntry * scenario.marketSentiment\n\n    // APY attractiveness: higher APY = more entries\n    apyAttractiveness = CLAMP(prevState.nominalApy / 0.5, 0.2, 3.0)\n    // ^^ normalized so 50% APY = 1.0 attractiveness\n\n    newUsers = ROUND(sentimentAdjustedEntry * apyAttractiveness)\n\n    // 3b. Calculate exit rate\n    // Base exit probability, increased by poor returns\n    exitProbBase = behavior.baseExitProbability\n\n    // If real APY is negative, exits increase\n    returnPenalty = 0\n    IF prevState.realApy < 0:\n      returnPenalty = ABS(prevState.realApy) * behavior.returnSensitivity\n\n    // If price is falling, exits increase (momentum-chasing behavior)\n    priceMomentum = 0\n    IF t >= 7:\n      priceChange7d = (prevState.tokenPrice - results[t-7].tokenPrice)\n                      / results[t-7].tokenPrice\n      IF priceChange7d < -0.05:  // more than 5% drop in 7 steps\n        priceMomentum = ABS(priceChange7d) * behavior.returnSensitivity * 0.5\n\n    effectiveExitProb = MIN(exitProbBase + returnPenalty + priceMomentum, 0.5)\n    // ^^ cap at 50% to prevent instant total collapse (unrealistic)\n\n    exitingUsers = ROUND(prevState.activeUsers * effectiveExitProb)\n\n    // 3c. Process shock events\n    FOR shock IN scenario.shockEvents WHERE shock.step == t:\n      SWITCH shock.type:\n        CASE 'mass-exit':\n          exitingUsers += ROUND(prevState.activeUsers * shock.magnitude)\n        CASE 'growth-halt':\n          newUsers = 0\n          // Set a flag to suppress growth for shock.magnitude steps\n        CASE 'price-crash':\n          // Applied in PHASE 5\n        CASE 'whale-entry':\n          // Add a large staker\n          newUsers += 1\n          // whale stake handled separately below\n\n    // 3d. Update user count\n    exitingUsers = MIN(exitingUsers, prevState.activeUsers)  // can't exit more than exist\n    state.activeUsers = prevState.activeUsers + newUsers - exitingUsers\n    state.activeUsers = MAX(state.activeUsers, 0)\n    state.newUsersThisStep = newUsers\n    state.exitedUsersThisStep = exitingUsers\n\n    // 3e. Update staking totals\n    newStake = newUsers * behavior.averageStakeSize\n    exitStake = exitingUsers * behavior.averageStakeSize\n\n    // Early unstake penalties\n    penaltyTokens = 0\n    IF params.lockPeriod > 0:\n      // Fraction of exiting users who are within lock period\n      // Simplified: assume uniform entry distribution\n      lockedFraction = MIN(params.lockPeriod / MAX(t, 1), 1.0)\n      penaltyTokens = exitStake * lockedFraction * params.earlyUnstakePenalty\n\n      SWITCH params.penaltyDestination:\n        CASE 'treasury':\n          state.treasuryBalanceTokens += penaltyTokens\n        CASE 'burn':\n          state.totalSupply -= penaltyTokens\n        CASE 'redistribute':\n          stakerRewards += penaltyTokens\n\n    state.totalStaked = prevState.totalStaked + newStake - exitStake\n    state.totalStaked = MAX(state.totalStaked, 0)\n\n    // ========================================\n    // PHASE 4: DISTRIBUTE REWARDS\n    // ========================================\n    \n    // Can treasury cover the staker rewards?\n    actualRewards = stakerRewards\n    IF actualRewards > state.treasuryBalanceTokens AND params.stakerEmissionShare < 1.0:\n      // If rewards come from treasury and it's depleted, rewards decrease\n      // (Only applies when rewards are paid from treasury, not freshly minted)\n      // For emission-funded rewards, they're always paid (newly minted)\n      // Treasury funds fee-redistribution rewards\n      pass  // emissions are newly minted, always available\n\n    state.rewardsDistributedThisStep = actualRewards\n    state.cumulativeRewards = prevState.cumulativeRewards + actualRewards\n\n    // Compounding: fraction of users re-stake their rewards\n    compoundedRewards = actualRewards * behavior.compoundingFraction\n    claimedRewards = actualRewards - compoundedRewards\n\n    state.totalStaked += compoundedRewards\n    state.circulatingSupply = state.totalSupply\n                              - state.treasuryBalanceTokens\n                              - state.totalStaked\n                              + claimedRewards\n    // ^^ Claimed rewards enter circulation; compounded stay staked\n\n    // ========================================\n    // PHASE 5: PRICE DISCOVERY\n    // ========================================\n    \n    IF params.priceModel == 'constant':\n      state.tokenPrice = params.initialTokenPrice\n\n    ELSE IF params.priceModel == 'supply-demand':\n      // Price is driven by net buy/sell pressure\n      \n      // Buy pressure: new stakers buying tokens + external demand\n      buyPressure = newStake * prevState.tokenPrice  // USD value of new stakes\n      IF params.externalBuyPressure:\n        buyPressure += params.externalBuyPressure\n\n      // Sell pressure: exiting stakers selling + claimed rewards being sold\n      sellPressure = exitStake * prevState.tokenPrice  // exiting stakers sell\n      // Assume 50% of claimed (non-compounded) rewards are sold\n      sellPressure += claimedRewards * prevState.tokenPrice * 0.5\n\n      // Net pressure determines price change\n      totalLiquidity = state.circulatingSupply * prevState.tokenPrice\n      netPressure = buyPressure - sellPressure\n\n      // Price impact = net_pressure / (elasticity * total_liquidity)\n      // Higher elasticity = less price impact per unit of pressure\n      priceImpact = netPressure / (params.priceElasticity * totalLiquidity + 1)\n      // ^^ +1 prevents division by zero\n\n      // Dilution pressure from supply increase\n      supplyInflation = emissions / prevState.totalSupply\n      dilutionPressure = -supplyInflation * 0.3\n      // ^^ 30% of inflation transmits to price (market doesn't fully price in)\n\n      // External pressure from scenario config\n      externalPressure = scenario.externalPricePressure - 1.0  // 1.0 = neutral\n\n      // Shock events\n      shockPressure = 0\n      FOR shock IN scenario.shockEvents WHERE shock.step == t:\n        IF shock.type == 'price-crash':\n          shockPressure = -shock.magnitude\n\n      // Apply all price forces\n      totalPriceChange = priceImpact + dilutionPressure + externalPressure + shockPressure\n\n      // Dampen to prevent wild swings (max 20% change per step)\n      totalPriceChange = CLAMP(totalPriceChange, -0.20, 0.20)\n\n      state.tokenPrice = prevState.tokenPrice * (1 + totalPriceChange)\n      state.tokenPrice = MAX(state.tokenPrice, 0.000001)  // floor at near-zero\n\n      state.buyPressureUsd = buyPressure\n      state.sellPressureUsd = sellPressure\n      state.netPressureUsd = netPressure\n\n    // ========================================\n    // PHASE 6: MEASURE\n    // ========================================\n    \n    state.stakingRatio = state.totalStaked / MAX(state.circulatingSupply, 1)\n    state.marketCap = state.tokenPrice * state.circulatingSupply\n    state.fullyDilutedValuation = state.tokenPrice * state.totalSupply\n\n    // Nominal APY: (daily_rewards / total_staked) * 365\n    dailyRewardRate = state.rewardsDistributedThisStep / MAX(state.totalStaked, 1)\n    state.nominalApy = annualize(dailyRewardRate, config.timeConfig.stepUnit)\n\n    // Real APY: accounts for token price change\n    IF t >= 1:\n      priceChangeRate = (state.tokenPrice - prevState.tokenPrice) / prevState.tokenPrice\n      state.realApy = annualize(dailyRewardRate + priceChangeRate, config.timeConfig.stepUnit)\n    ELSE:\n      state.realApy = state.nominalApy\n\n    // Effective APY: what you actually earn in USD\n    state.effectiveApy = state.realApy  // same as real for daily resolution\n\n    // Treasury metrics\n    state.treasuryBalanceUsd = state.treasuryBalanceTokens * state.tokenPrice\n    state.feeRevenueThisStep = feeRevenue\n    state.cumulativeFeeRevenue = prevState.cumulativeFeeRevenue + feeRevenue\n\n    // Treasury runway\n    netTreasuryBurn = state.rewardsDistributedThisStep - feeRevenue - treasuryEmissions\n    IF netTreasuryBurn > 0:\n      state.treasuryRunwaySteps = state.treasuryBalanceTokens / netTreasuryBurn\n    ELSE:\n      state.treasuryRunwaySteps = Infinity  // treasury is growing\n\n    // Fee coverage ratio (rolling 30-step window)\n    feeWindow.push(feeRevenue)\n    rewardWindow.push(state.rewardsDistributedThisStep)\n    IF feeWindow.length > WINDOW_SIZE:\n      feeWindow.shift()\n      rewardWindow.shift()\n    totalFees = SUM(feeWindow)\n    totalRewards = SUM(rewardWindow)\n    state.feeCoverageRatio = totalFees / MAX(totalRewards, 0.001)\n\n    // Sustainability index (composite 0-1)\n    state.rewardSustainabilityIndex = computeSustainabilityIndex(state)\n\n    state.exitPressure = sellPressure\n\n    // Emission rate annualized\n    state.emissionsThisStep = emissions\n    state.emissionRateAnnualized = annualize(emissions / MAX(prevState.totalSupply, 1),\n                                              config.timeConfig.stepUnit)\n\n    // ========================================\n    // PHASE 7: SNAPSHOT\n    // ========================================\n    results.push(deepCopy(state))\n\n    // --- AGENT TRACKING (if enabled) ---\n    IF config.outputFlags.computeAgentLevelPnL:\n      updateAgents(agents, state, newUsers, exitingUsers, behavior)\n\n  END FOR\n\n  // --- POST-SIMULATION ---\n  riskClassification = classifyRisk(results, agents, config)\n\n  RETURN {\n    config: config,\n    timeSeries: results,\n    riskClassification: riskClassification,\n    agentResults: config.outputFlags.computeAgentLevelPnL\n                  ? computeAgentResults(agents) : undefined,\n    meta: { executionTimeMs: elapsed(), ... }\n  }\n\nEND FUNCTION\n```\n\n### 2.3 Emission Calculation Sub-Function\n\n```\nFUNCTION calculateEmissions(params, step, currentSupply): number\n  SWITCH params.emissionModel:\n    CASE 'fixed-rate':\n      RETURN params.fixedEmissionRate\n\n    CASE 'halving':\n      halvings = FLOOR(step / params.halvingInterval)\n      RETURN params.halvingInitialRate / (2 ^ halvings)\n\n    CASE 'decay':\n      RETURN params.decayInitialRate * EXP(-params.decayConstant * step)\n\n    CASE 'custom-schedule':\n      IF step < params.customSchedule.length:\n        RETURN params.customSchedule[step]\n      RETURN 0  // no more emissions after schedule ends\nEND FUNCTION\n```\n\n### 2.4 Entry Rate Calculation Sub-Function\n\n```\nFUNCTION calculateEntryRate(behavior, step): number\n  SWITCH behavior.entryModel:\n    CASE 'constant':\n      RETURN behavior.baseEntryRate\n\n    CASE 'linear-growth':\n      RETURN behavior.baseEntryRate * (1 + behavior.entryCoefficient * step)\n\n    CASE 'exponential-decay':\n      // Models: initial hype fades\n      RETURN behavior.baseEntryRate * EXP(-behavior.entryCoefficient * step)\n\n    CASE 's-curve':\n      // Logistic growth: slow start, rapid middle, saturation\n      midpoint = 60  // peak growth at step 60\n      RETURN behavior.baseEntryRate *\n             (4 / (1 + EXP(-behavior.entryCoefficient * (step - midpoint))))\n      // ^^ peaks at 4x base rate, then saturates\nEND FUNCTION\n```\n\n### 2.5 Annualization Helper\n\n```\nFUNCTION annualize(perStepRate, stepUnit): number\n  SWITCH stepUnit:\n    CASE 'hour':\n      RETURN (1 + perStepRate) ^ 8760 - 1    // 8760 hours/year\n    CASE 'day':\n      RETURN (1 + perStepRate) ^ 365 - 1\n    CASE 'week':\n      RETURN (1 + perStepRate) ^ 52 - 1\nEND FUNCTION\n```\n\n### 2.6 Agent Update Sub-Function\n\n```\nFUNCTION updateAgents(agents, state, newUsers, exitingUsers, behavior):\n  // Add new agents\n  FOR i = 0 TO newUsers - 1:\n    stake = sampleNormal(behavior.averageStakeSize, behavior.stakeDeviation)\n    stake = MAX(stake, 1)  // minimum 1 token\n    agents.push({\n      id: agents.length,\n      entryStep: state.step,\n      exitStep: null,\n      stakeAmount: stake,\n      stakeValueAtEntry: stake * state.tokenPrice,\n      cumulativeRewards: 0,\n      compounding: RANDOM() < behavior.compoundingFraction,\n      currentUnrealizedPnlUsd: 0,\n      currentUnrealizedPnlPercent: 0,\n      realized: false,\n      realizedPnlUsd: 0\n    })\n\n  // Distribute rewards proportionally to active agents\n  activeAgents = agents.filter(a => a.exitStep == null)\n  totalActiveStake = SUM(activeAgents.map(a => a.stakeAmount))\n\n  FOR EACH agent IN activeAgents:\n    share = agent.stakeAmount / MAX(totalActiveStake, 1)\n    reward = state.rewardsDistributedThisStep * share\n    IF agent.compounding:\n      agent.stakeAmount += reward\n    agent.cumulativeRewards += reward\n\n    // Update P&L\n    currentValue = agent.stakeAmount * state.tokenPrice\n                   + (agent.compounding ? 0 : agent.cumulativeRewards * state.tokenPrice)\n    agent.currentUnrealizedPnlUsd = currentValue - agent.stakeValueAtEntry\n    agent.currentUnrealizedPnlPercent = agent.currentUnrealizedPnlUsd / agent.stakeValueAtEntry\n\n  // Exit agents (those with worst returns exit first -- behavioral realism)\n  activeAgents.sort(BY currentUnrealizedPnlPercent ASCENDING)\n  FOR i = 0 TO MIN(exitingUsers, activeAgents.length) - 1:\n    agent = activeAgents[i]\n    agent.exitStep = state.step\n    agent.realized = true\n    finalValue = agent.stakeAmount * state.tokenPrice\n                 + agent.cumulativeRewards * state.tokenPrice\n    agent.realizedPnlUsd = finalValue - agent.stakeValueAtEntry\n\nEND FUNCTION\n```\n\n### 2.7 Death Spiral Detection\n\nThe simulation naturally models death spirals through the feedback loop:\n\n```\nPrice drops --> Real APY goes negative --> Exit probability increases -->\nMore users exit --> Sell pressure increases --> Price drops further -->\nTreasury value falls --> Runway shortens --> More exits ...\n```\n\nThe engine does not need special \"death spiral\" code. The behavioral model's `returnSensitivity` parameter controls how quickly negative returns trigger exits. At high sensitivity values (0.8-1.0), death spirals emerge naturally from the simulation dynamics. The risk engine detects this pattern post-hoc by looking for accelerating exit rates combined with declining price.\n\n### 2.8 Determinism and Reproducibility\n\nThe simulation uses a seeded PRNG (e.g., `xorshift128+` or similar) for all stochastic elements (user stake size sampling, individual exit decisions). The seed is derived from `config.id`. This ensures that identical configs always produce identical results, which is critical for:\n- Consistent slider behavior (no jitter when parameters do not change)\n- Reproducible scenario comparisons\n- Testability\n\n---\n\n## 3. Risk Engine Design\n\n### 3.1 Classification Algorithm\n\nThe risk classification runs after the full simulation completes. It examines the entire `SimulationState[]` time series to compute aggregate metrics, then applies a rule-based classifier.\n\n```\nFUNCTION classifyRisk(timeSeries, agents, config): RiskClassification\n\n  T = timeSeries.length\n\n  // ========================================\n  // METRIC 1: Fee Coverage Ratio (Primary Signal)\n  // ========================================\n  // Average fee coverage over the simulation's latter half\n  // (ignore early steps where treasury bootstrapping distorts the ratio)\n  latterHalf = timeSeries.slice(FLOOR(T / 2))\n  totalFees = SUM(latterHalf.map(s => s.feeRevenueThisStep))\n  totalRewards = SUM(latterHalf.map(s => s.rewardsDistributedThisStep))\n  avgFeeCoverage = totalFees / MAX(totalRewards, 0.001)\n\n  // ========================================\n  // METRIC 2: Treasury Trajectory\n  // ========================================\n  // Is the treasury growing, stable, or depleting?\n  treasuryStart = timeSeries[FLOOR(T / 4)].treasuryBalanceTokens  // skip initial\n  treasuryEnd = timeSeries[T - 1].treasuryBalanceTokens\n  treasuryTrend = (treasuryEnd - treasuryStart) / MAX(treasuryStart, 1)\n  // Positive = growing, negative = depleting\n\n  treasuryDepleted = timeSeries.some(s => s.treasuryBalanceTokens <= 0)\n  treasuryDepletionStep = treasuryDepleted\n    ? timeSeries.findIndex(s => s.treasuryBalanceTokens <= 0)\n    : null\n\n  // ========================================\n  // METRIC 3: New Deposit Dependency\n  // ========================================\n  // Counterfactual: what fraction of reward funding comes from\n  // new deposits vs organic fee revenue?\n  // \n  // Method: Run a shadow calculation where we remove new user inflows\n  // and measure how quickly the system collapses.\n  //\n  // Approximation: (rewards - fees) / rewards = dependency on non-fee sources\n  nonFeeFunding = MAX(totalRewards - totalFees, 0)\n  newDepositDependency = nonFeeFunding / MAX(totalRewards, 0.001)\n\n  // ========================================\n  // METRIC 4: Return Fairness (Gini Coefficient)\n  // ========================================\n  gini = 0\n  earlyVsLateRatio = 1\n  IF agents AND agents.length > 0:\n    returns = agents.map(a => a.realized ? a.realizedPnlPercent : a.currentUnrealizedPnlPercent)\n    gini = computeGini(returns)\n\n    // Cohort comparison\n    earlyAgents = agents.filter(a => a.entryStep < T / 3)\n    lateAgents = agents.filter(a => a.entryStep >= 2 * T / 3)\n    earlyAvg = MEAN(earlyAgents.map(a => returnOf(a)))\n    lateAvg = MEAN(lateAgents.map(a => returnOf(a)))\n    earlyVsLateRatio = earlyAvg / MIN(lateAvg, -0.001)  // handle near-zero\n\n  // ========================================\n  // METRIC 5: Price Stability\n  // ========================================\n  prices = timeSeries.map(s => s.tokenPrice)\n  maxPrice = MAX(prices)\n  minPriceAfterMax = MIN(prices.slice(prices.indexOf(maxPrice)))\n  maxDrawdown = (maxPrice - minPriceAfterMax) / maxPrice\n\n  realApyNegativeStep = timeSeries.findIndex(s => s.realApy < 0)\n  IF realApyNegativeStep == -1: realApyNegativeStep = null\n\n  // ========================================\n  // METRIC 6: Stress Test Pass Rate\n  // ========================================\n  stressResults = runStressTests(config)  // see Section 7\n  passRate = stressResults.filter(r => r.passed).length / stressResults.length\n\n  // ========================================\n  // CLASSIFICATION RULES\n  // ========================================\n  \n  scores = {\n    feeCoverageRatio: avgFeeCoverage,\n    treasuryRunway: timeSeries[T-1].treasuryRunwaySteps,\n    newDepositDependency: newDepositDependency,\n    returnGini: gini,\n    maxPriceDrawdown: maxDrawdown,\n    realApyNegativeStep: realApyNegativeStep,\n    earlyVsLateReturnRatio: earlyVsLateRatio,\n    stressTestPassRate: passRate\n  }\n\n  // Primary classification: fee coverage ratio\n  IF avgFeeCoverage >= 0.80:\n    rating = 'sustainable'\n    confidence = MIN(avgFeeCoverage / 1.0, 1.0)  // 100% coverage = full confidence\n  ELSE IF avgFeeCoverage >= 0.30:\n    rating = 'caution'\n    confidence = 1.0 - ABS(avgFeeCoverage - 0.55) / 0.25\n    // ^^ most confident at the middle of the range\n  ELSE:\n    rating = 'ponzi-structure'\n    confidence = MIN((0.30 - avgFeeCoverage) / 0.30, 1.0)\n\n  // Override rules: certain conditions force a downgrade\n  IF rating == 'sustainable':\n    IF treasuryDepleted:\n      rating = 'caution'  // can't be sustainable if treasury ever hits zero\n    IF maxDrawdown > 0.7:\n      rating = 'caution'  // 70%+ drawdown is not sustainable behavior\n    IF gini > 0.8:\n      rating = 'caution'  // extreme return inequality\n\n  IF rating == 'caution':\n    IF treasuryDepleted AND treasuryDepletionStep < T / 2:\n      rating = 'ponzi-structure'  // depletes in first half = structural problem\n    IF newDepositDependency > 0.9 AND maxDrawdown > 0.5:\n      rating = 'ponzi-structure'  // almost entirely deposit-funded + crash\n\n  // ========================================\n  // GENERATE REPORT\n  // ========================================\n  report = generateReport(rating, scores, timeSeries, agents)\n  criticalTimesteps = findCriticalTimesteps(timeSeries)\n\n  RETURN {\n    rating: rating,\n    confidence: confidence,\n    scores: scores,\n    report: report,\n    criticalTimesteps: criticalTimesteps\n  }\n\nEND FUNCTION\n```\n\n### 3.2 Gini Coefficient Computation\n\n```\nFUNCTION computeGini(values: number[]): number\n  // Standard Gini coefficient calculation\n  // 0 = perfect equality, 1 = perfect inequality\n  n = values.length\n  IF n == 0: RETURN 0\n\n  sorted = values.sort(ASCENDING)\n  cumulativeSum = 0\n  weightedSum = 0\n\n  FOR i = 0 TO n - 1:\n    cumulativeSum += sorted[i]\n    weightedSum += (2 * (i + 1) - n - 1) * sorted[i]\n\n  IF cumulativeSum == 0: RETURN 0\n  RETURN weightedSum / (n * cumulativeSum)\nEND FUNCTION\n```\n\n### 3.3 Report Generation Strategy\n\nThe report is generated using a **template-based system with conditional logic** rather than AI generation. This ensures determinism, speed, and auditability. The templates are rich enough to produce natural-sounding English.\n\n```typescript\n/**\n * Report generation uses a structured template system.\n * Each finding has a template that interpolates computed values.\n * The headline and summary use the rating + scores to select\n * from a bank of pre-written templates.\n */\n\n// HEADLINE TEMPLATES (indexed by rating)\nconst HEADLINES: Record<RiskClassification['rating'], string[]> = {\n  'sustainable': [\n    'This protocol generates real yield from fees.',\n    'Protocol revenue covers staking rewards sustainably.',\n  ],\n  'caution': [\n    'This protocol partially relies on new deposits. Performance Architecture\n\n### 5.1 The Performance Challenge\n\nThe target: a user drags a slider, and the chart updates within one frame (16ms). But a full 180-step simulation with 500 agents involves tens of thousands of arithmetic operations. This requires a layered performance strategy.\n\n### 5.2 Three-Tier Execution Model\n\n```\nTIER 1: FAST PATH (< 5ms, runs on main thread)\n  - Used during slider drags\n  - Pre-computes a sensitivity matrix at initialization\n  - When a single parameter changes, interpolates the result\n    rather than re-running the full simulation\n  - Only updates the primary chart series (price, APY, treasury)\n  - Skips agent-level computation entirely\n  - Uses cached intermediate values\n\nTIER 2: FULL AGGREGATE (< 50ms, runs in Web Worker)\n  - Used when slider drag ends (mouseup/touchend)\n  - Runs the complete simulation loop with aggregate user model\n  - No individual agent tracking\n  - Updates all charts and metrics\n  - Runs asynchronously; result replaces the fast-path approximation\n\nTIER 3: FULL AGENT-BASED (< 500ms, runs in Web Worker)\n  - Used on initial load, explicit \"Run Simulation\" button, and scenario comparison\n  - Runs complete simulation with individual agent tracking\n  - Computes Gini coefficient, cohort analysis, P&L distributions\n  - Updates risk classification and generates full report\n  - Can be further parallelized across multiple workers for scenario batches\n```\n\n### 5.3 Fast Path: Sensitivity Matrix Pre-computation\n\n```\nCONCEPT:\n  At initialization (Tier 3 run), we also compute how each output\n  varies with respect to each slider parameter. This produces a\n  Jacobian-like sensitivity matrix.\n\n  For parameter p_i and output o_j:\n    sensitivity[i][j] = d(o_j) / d(p_i)\n  \n  Evaluated at the current parameter values using finite differences:\n    sensitivity[i][j] ≈ (simulate(p_i + delta) - simulate(p_i - delta)) / (2 * delta)\n  \n  This is expensive (2 * numParams extra simulations), but only done once\n  after each Tier 3 run.\n\nDURING SLIDER DRAG:\n  When parameter p_i changes by amount dp:\n    For each time step t:\n      For each output o_j:\n        o_j[t] ≈ o_j_baseline[t] + sensitivity[i][j][t] * dp\n\n  This is a simple vector addition: O(steps * outputs) ≈ O(180 * 10) = O(1800)\n  operations. Well under 1ms.\n\nLIMITATION:\n  Linear approximation degrades for large parameter changes.\n  The engine tracks cumulative delta from the baseline and triggers\n  a Tier 2 re-simulation when delta exceeds a threshold (e.g., >20%\n  change from the baseline value).\n```\n\n### 5.4 Web Worker Architecture\n\n```typescript\n/**\n * The simulation runs in a dedicated Web Worker to keep the UI thread free.\n * Communication is via structured message passing.\n */\n\n// --- Messages from Main Thread to Worker ---\ntype WorkerInMessage =\n  | { type: 'RUN_FULL'; config: SimulationConfig }\n  | { type: 'RUN_FAST_PATH'; paramIndex: number; delta: number; baseline: SimulationResult }\n  | { type: 'ABORT' };\n\n// --- Messages from Worker to Main Thread ---\ntype WorkerOutMessage =\n  | { type: 'RESULT'; result: SimulationResult }\n  | { type: 'FAST_PATH_RESULT'; timeSeries: SimulationState[] }\n  | { type: 'PROGRESS'; step: number; totalSteps: number }\n  | { type: 'ERROR'; message: string };\n\n/**\n * WorkerPool manages a pool of workers for parallel scenario execution.\n * For slider interactions, a single dedicated worker is used.\n * For scenario comparison (3+ scenarios), multiple workers run in parallel.\n */\nclass WorkerPool {\n  private workers: Worker[] = [];\n  private readonly poolSize: number;\n\n  constructor(poolSize: number = navigator.hardwareConcurrency || 4) {\n    this.poolSize = Math.min(poolSize, 4); // cap at 4 to be respectful of resources\n  }\n\n  /**\n   * Run multiple scenarios in parallel, returning results as they complete.\n   */\n  async runScenarios(\n    baseConfig: SimulationConfig,\n    scenarios: ScenarioConfig[]\n  ): Promise<SimulationResult[]> {\n    // Distribute scenarios across workers round-robin\n    // Each worker gets a config with a different scenario\n    // Results are collected and returned in order\n  }\n}\n```\n\n### 5.5 WASM Consideration\n\nFor the MVP, pure TypeScript in a Web Worker is sufficient. The simulation loop is arithmetic-heavy but not massively parallel. Based on benchmarks from the research phase, WASM offers approximately 2x speedup over optimized TypeScript for this class of computation. The recommendation:\n\n- **MVP**: TypeScript in Web Worker. Target 180 steps, 500 agents, < 100ms. Achievable without WASM.\n- **Phase 2 optimization**: If composite models or 1000+ agents push execution past 200ms, compile the inner simulation loop to WASM using AssemblyScript (TypeScript-like syntax, compiles to WASM). The interfaces remain identical; only the inner loop changes.\n- **Phase 3 (if needed)**: Rust-compiled WASM with SIMD for maximum throughput. This is the nuclear option and unlikely to be needed for client-side simulations of this scale.\n\n### 5.6 Memory Considerations\n\nA `SimulationState` has approximately 30 numeric fields (8 bytes each) = ~240 bytes per state. For 180 steps, that is ~43 KB per simulation run. With 500 `AgentState` objects (~128 bytes each) over 180 steps of updates, agent tracking adds ~12 MB of peak memory. This is well within browser limits.\n\nFor scenario comparison with 5 scenarios, peak memory is approximately 60 MB. Manageable, but the engine should use `transferable` objects when posting results from workers to avoid copying.\n\n---\n\n## 6.
